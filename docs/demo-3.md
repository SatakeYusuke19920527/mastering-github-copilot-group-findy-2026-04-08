# DEMO 3: ビジネス職向け — 競合調査から報告メモまで

## 概要

コードを一切書かずに、GitHub Copilot CLI だけで**学習塾の競合調査→要約→打ち手立案→リスク検証→報告メモ作成**を完結させるデモ。
ビジネス職（経営企画・マーケ・マネージャー）が Copilot を業務で活用できることを示す。

---

## 対象者

- エンジニア以外のビジネス職
- 「GitHub Copilot = コーディング支援」と思っている人

## メッセージ

> Copilot はコードだけじゃない。調査・分析・意思決定支援・ドキュメント作成まで、ビジネスの思考パートナーになる。

---

## DEMOフロー

### Step 1: `/research` — 競合調査

```
Researching: 学習塾業界の競合調査をしてほしい。以下の観点で調べて：

1. 個別指導塾の主要プレイヤー（明光義塾、個別教室のトライ、森塾、ITTO個別指導学院など）
2. 各社の強み・差別化ポイント（料金体系、AI活用、オンライン対応）
3. 2024-2026年の業界トレンド（少子化対応、EdTech、AI教材）
4. 小規模塾が大手に勝てるポイント
```

**見せどころ**: Copilot が Web を横断して最新情報を収集・整理する様子

---

### Step 2: 調査結果の要約

```
今の調査結果を、以下のフォーマットで要約して：

■ 市場概況（3行以内）
■ 主要競合の比較表（社名・生徒数・強み・弱み・月謝目安）
■ 業界トレンド TOP3
■ 小規模塾にとっての機会と脅威
```

**見せどころ**: 大量の調査結果を構造化された要約に即座に変換

---

### Step 3: 自塾の打ち手を考えさせる

```
あなたは学習塾の経営コンサルタントです。
以下の前提で、佐竹塾（個人経営・生徒50名・中学生中心）の成長戦略を5つ提案してください。

前提条件:
- 所在地: 東京都郊外
- 強み: 講師の質が高い、保護者との距離が近い
- 課題: 認知度が低い、デジタル化が遅れている
- 予算: 年間マーケ費 100万円以内

各提案には「施策名」「概要」「想定コスト」「期待効果」を含めて。
```

**見せどころ**: ビジネスコンテキストを与えると、具体的かつ実行可能な施策を提案

---

### Step 4: 反論・リスク分析

```
今提案した5つの打ち手それぞれに対して、「悪魔の代弁者」として反論してください。

各施策について:
- 最大のリスクは何か
- 失敗するとしたらどんなシナリオか
- 競合が同じことをしたらどうなるか
- それでも実行すべき理由（あれば）

最後に、リスクを踏まえた上での優先順位（おすすめ順）をつけて。
```

**見せどころ**: 自分で考えた施策に自分でツッコミを入れさせる「壁打ち」活用法

---

### Step 5: 上司・チーム向け報告メモに変換

```
ここまでの調査・分析・提案を、上司への報告メモ（A4 1枚相当）にまとめてください。

フォーマット:
━━━━━━━━━━━━━━━━━━━━
件名: 学習塾市場の競合調査と成長戦略の提案
作成日: 2026年4月8日
作成者: 経営企画部

1. 調査目的（2行）
2. 市場概況（3行）
3. 競合比較（表形式）
4. 提案施策（優先順位付き、各2行）
5. リスクと対応方針（箇条書き）
6. 次のアクション
━━━━━━━━━━━━━━━━━━━━

ビジネス文書として丁寧な日本語で。
```

**見せどころ**: 分析結果を「上司が読める形」に一発変換。フォーマット指定で出力を制御。

---

## デモのキーポイント

### 🎯 ビジネス職への訴求ポイント

| # | ポイント | 説明 |
|---|---------|------|
| 1 | **調査時間の圧縮** | 数時間かかる競合調査を数分で完了 |
| 2 | **思考の壁打ち相手** | 自分のアイデアに反論させて質を上げる |
| 3 | **フォーマット変換** | 同じ内容を「報告書」「提案書」「メール」に即変換 |
| 4 | **コンテキスト維持** | 会話の流れで深掘り・方向転換が自在 |

### 💡 デモで強調すべきこと

- **コードは一行も書いていない** — ターミナルだけで完結
- **プロンプトの書き方がスキル** — 具体的な指示ほど良い出力になる
- **チーム全員が使える** — エンジニアだけのツールではない
- **GitHub Issues との連携** — 施策をそのままタスク管理に落とせる（応用編として紹介可）

## 想定時間

| Step | 内容 | 目安 |
|------|------|------|
| Step 1 | `/research` 競合調査 | 2-3分 |
| Step 2 | 要約 | 1分 |
| Step 3 | 打ち手提案 | 1-2分 |
| Step 4 | 反論・リスク | 1-2分 |
| Step 5 | 報告メモ変換 | 1分 |
| **合計** | | **6-9分** |
